1. The attacker compromises the system and drops the GlassRAT dropper component signed with a stolen certificate. 2. The dropper installs the GlassRAT DLL payload to the ProgramData folder and executes it using rundll32. 3. The DLL establishes command and control communication to remote servers. 4. The attacker issues commands to the implanted GlassRAT for surveillance and data exfiltration. 5. GlassRAT can respond with system info, reverse shell sessions, file transfers, command execution, etc. 6. The malware persists via registry Run keys or service installation depending on privileges. 7. Network connections and behaviors are observable even if the malware is not detected by antivirus products initially.
