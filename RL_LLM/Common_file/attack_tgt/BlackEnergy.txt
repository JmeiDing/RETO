1. The attack starts with a malicious Excel spreadsheet containing a macro. 2. The macro assembles the BlackEnergy executable code and saves it as vba_macro.exe in the temp directory. 3. vba_macro.exe executes and drops additional modules: - FONTCACHE.DAT - a network sniffer module - rundll32.exe - a legitimate Windows binary - NTUSER.LOG - an empty file - desktop.ini - a default ini file 4. vba_macro.exe executes FONTCACHE.DAT using rundll32.exe and passes it the network adapter to hook. 5. FONTCACHE.DAT starts capturing network packets and sends system info like keyboard layout to a C2 server. 6. FONTCACHE.DAT may also open a backdoor for remote access and data exfiltration. 7. vba_macro.exe registers itself to auto-start and then deletes itself. 8. The network sniffer FONTCACHE.DAT persists on the system gathering data and communicating with the C2.
