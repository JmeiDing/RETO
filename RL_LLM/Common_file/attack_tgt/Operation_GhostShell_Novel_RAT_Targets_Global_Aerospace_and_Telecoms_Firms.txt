 1. Initial compromise vectors are unknown, but likely start with phishing emails or exploits. 2. The ShellClient RAT is installed on victim machines, often disguised as a svchost process. 3. ShellClient gathers system info and fingerprints the machine to create a unique identifier. 4. The malware connects to command and control servers, originally Azure domains, now Dropbox accounts. 5. Commands are fetched from Dropbox to perform reconnaissance, lateral movement, and data exfiltration. 6. Credentials are dumped using tools like SafetyKatz. Data is archived and stolen via the C2. 7. New payloads can be downloaded over time to maintain persistence using Windows services. The RAT continues communicating with Dropbox C2 to receive updated commands.
