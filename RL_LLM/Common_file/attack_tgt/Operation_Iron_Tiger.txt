 1. The attack starts with a spear-phishing email containing a malicious document. 2. If the document is opened, it installs a custom downloader malware called Stage 1. 3. Stage 1 downloads and executes the main backdoor malware called Stage 2. 4. Stage 2 connects to command servers over HTTPS using a custom protocol to exchange encryption keys. 5. Stage 2 sends requests to the server to receive and execute commands for gathering system info, downloading modules, exfiltrating files, etc. 6. Stolen data is archived and exfiltrated to the attackers' servers. 7. Additional tools like PowerShell scripts and password dumpers are used for lateral movement and credential theft.
