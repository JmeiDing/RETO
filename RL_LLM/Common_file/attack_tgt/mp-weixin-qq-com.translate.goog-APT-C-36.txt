1. The attacker sends a spear phishing email containing a PDF attachment to the target. 2. The PDF contains a link that downloads a RAR archive file. 3. The RAR file is password protected and contains a VBS script with a PDF icon and file name. 4. The user is tricked into executing the VBS script. 5. The VBS script contains obfuscated code that reflects DLLs into memory. 6. The DLLs download and inject the final RAT payload into a signed process like RegAsm. 7. The RAT connects to a C2 server to enable remote control of the infected system. 8. The RAT can leverage various modules for functionality like keylogging, remote desktop, etc. 9. The attacker leverages the RAT to further exploit the target network.
