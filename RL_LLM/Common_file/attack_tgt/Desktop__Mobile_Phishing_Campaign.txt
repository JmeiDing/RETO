1. Attackers inject malicious JavaScript code into legitimate South Korean websites. 2. The script checks the HTTP referer header to validate visitors are real users from search engines or social media. 3. It checks the user-agent string to identify if the visitor is on desktop or mobile. 4. For mobile users, a fake login pop-up appears after the 6th site visit and button click. 5. For desktop users, the fake login appears immediately on top of the website. 6. The fake login prompts for username and password, claiming the site is unavailable without logging in. 7. Users enter their credentials which are sent to the attacker's server without validation. 8. The phishing login expires after 12 hours to avoid detection. 9. Attackers continuously evolve the code and infrastructure to avoid takedowns.
