1. The attack starts with a spear-phishing email containing a weaponized document exploiting CVE-2015-2545. 2. If the document is opened, it drops and executes the TidePool malware DLL. 3. TidePool establishes persistence via an ActiveSetup registry key to execute on boot. 4. The malware gathers system information and exfiltrates it to a C2 server via HTTP. 5. TidePool can receive commands from the C2 server to read, write, delete files, run commands, etc. 6. Stolen data is archived and exfiltrated to the attacker's servers. 7. The malware moves laterally through compromised networks.
