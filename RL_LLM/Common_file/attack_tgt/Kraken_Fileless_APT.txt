1. The attack starts with a phishing email containing a weaponized Word document. 2. The document contains a malicious macro that uses CactusTorch to load and execute the Kraken .NET loader. 3. The Kraken loader injects shellcode into the WerFault.exe process. 4. The shellcode uses anti-analysis checks to detect sandboxes and debugging. 5. The shellcode makes an HTTP request to download another payload from a compromised server. 6. The downloaded payload is injected into a process for execution.
