1. The threat actor gains initial access by exploiting vulnerabilities to deploy webshells on the target web server. 2. Discovery activities are run to enumerate system details like users, network connections, etc. 3. Credentials are dumped from the system using a variety of tools like Mimikatz. 4. LOLBins like Regsvr32 and MSIExec are used to bypass defenses and achieve command execution. 5. The Explosive RAT malware is deployed on the system as the final payload. 6. Explosive RAT contacts the C2 server and can execute commands uploaded by the threat actor. 7. The actor continues to move laterally through the network and repeat the process to obtain more credentials and compromise additional systems.
