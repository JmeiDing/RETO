1. The attacker sends a spearphishing email with a malicious attachment customized for the target. 2. If the user opens the attachment, the launcher program is installed. 3. The launcher program unpacks and extracts multiple components. 4. One component downloads the RAT payload from a command and control server. 5. The RAT payload runs in memory without touching disk. 6. The RAT connects to the C2 server and can receive commands. 7. It begins keylogging and exfiltrating data from the infected machine back to the C2 server. 8. The attacker can interactively issue commands to further control and steal data from the target.
