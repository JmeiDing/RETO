1. The threat actors send spear phishing emails with weaponized attachments to targets. 2. When the attachment is opened, it drops a decoy media file and a downloader. 3. The downloader retrieves the main malware payload disguised as a Flash file. 4. The payload is decoded and dropped onto the system, named to emulate legitimate software. 5. The malware connects to a C2 server, beaconing over HTTPS with encrypted cookie data. 6. The C2 server sends commands in JSON format for the malware to execute on the infected system. 7. Available commands allow the threat actors to remotely control the system, download/upload files, execute arbitrary commands, and exfiltrate data.
