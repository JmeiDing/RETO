1. The threat actor sends spearphishing emails with malicious PDF attachments. 2. The victim opens the PDF, triggering the exploit. 3. The dropper installs the main DLL component on the system. 4. The main DLL decrypts the hardcoded Twitter account information. 5. The malware tweets the C2 server which responds with updated modules or payloads. 6. The malware downloads the updates packaged as encrypted GIF files. 7. The malware decrypts and executes the payloads, which can include remote access tools. 8. The remote access payload enables the threat actor to control the infected system, exfiltrate data, etc.
