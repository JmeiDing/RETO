 1. The malware is delivered via a remote exploitation of a custom URL scheme vulnerability. 2. It establishes persistence by installing itself as a login item to auto-start on boot. 3. On first run, it generates a unique ID and saves it to a file. 4. The malware connects to hardcoded C2 servers to receive commands. 5. It can execute received commands, exfiltrate files, etc. based on C2 instructions. 6. The malware copies itself to the user's Library folder and executes that copy. 7. It decrypts strings related to C2 URLs and file extensions using a hardcoded AES key.
