 1. The attackers compromise the website of the Polish Financial Supervision Authority (KNF) by modifying a JavaScript file to load an external malicious script. 2. The malicious script executes when KNF website visitors browse to the compromised page, loading malware onto their machines. 3. The malware connects to attacker-controlled servers to download additional payloads. 4. Executed malware performs reconnaissance on the victim bank's network. 5. Malware moves laterally to compromise additional systems inside the banks. 6. Sensitive data is identified and exfiltrated back to the attacker's servers. 7. Exfiltrated data is encrypted to avoid detection during the theft. 8. Banks detect unusual network activity and unknown files on systems. 9. Forensic investigation reveals traces of the attack cycle.
